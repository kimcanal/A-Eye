# Module 1 Code Map

This document explains, in simple terms, what each Module 1 file does.

## `module1_simulation/minimal_simulation.py`
This is the small runnable simulation.
- It creates the city scenario.
- It places the required people, taxis, vehicles, and obstacles.
- It moves taxis and vehicles for a few time steps.
- It writes the results to JSON files.

### Simple explanation
Think of it as a tiny fake city that can move for a few seconds and write down what happened.

## `module1_simulation/minimal_stub.py`
This is the static structure file.
- It describes the scenario as data objects.
- It is useful as a clean template for the simulation settings.

### Simple explanation
Think of it as the blueprint or form that says how many passengers, taxis, and obstacles should exist.

## `module1_simulation/README.md`
This file explains how to run the module and what outputs are created.

### Simple explanation
Think of it as the user manual for Module 1.

## `module1_simulation/scenario.md`
This file explains the minimum scenario in plain language.

### Simple explanation
Think of it as the story of the small city.

## `module1_simulation/asset_requirements.md`
This file lists what kind of 3D assets are needed.

### Simple explanation
Think of it as the shopping list for the Unity scene.

## `module1_simulation/integration_plan.md`
This file explains how the simple scaffold can be connected to the real digital twin assets.

### Simple explanation
Think of it as the bridge between the fake small city and the real 3D city.

## `module1_simulation/assets/README.md`
This folder is reserved for the actual imported assets.

### Simple explanation
Think of it as the box where the 3D parts will go.
