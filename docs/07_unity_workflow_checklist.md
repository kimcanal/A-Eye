# Unity Workflow Checklist for Module 1

## Phase 1: Project setup
- [ ] Create a Unity project
- [ ] Set up a clean folder structure
- [ ] Confirm Unity version compatibility with the provided `.unitypackage` files

## Phase 2: Asset import
- [ ] Import `MapSetting_2.unitypackage`
- [ ] Import `street_main.unitypackage`
- [ ] Import `car_taxi.unitypackage`
- [ ] Import `obs_car_1.unitypackage`
- [ ] Import `obs_car_2.unitypackage`
- [ ] Import `obs_kickboard.unitypackage`
- [ ] Import `carwheel_move.unitypackage`
- [ ] Import `kickboard_fall.unitypackage`

## Phase 3: Scene composition
- [ ] Build the 3x3 block city scene
- [ ] Place the taxi object(s)
- [ ] Place obstacle objects
- [ ] Place passenger markers or proxies
- [ ] Validate the scene layout visually

## Phase 4: Behavior hookup
- [ ] Connect object movement to the scenario definition
- [ ] Trigger animation assets for motion events
- [ ] Record movement or call events
- [ ] Export a scene log for later analysis

## Phase 5: Verification
- [ ] Confirm the scene matches the assignment's Module 1 minimum configuration
- [ ] Confirm the Python scaffold and Unity scene describe the same scenario
- [ ] Confirm the outputs can be used as evidence of execution

## Notes
This checklist intentionally starts from the assets and works toward the final digital twin scene. It gives the team a concrete path from the downloaded Unity packages to a usable Module 1 environment.
