Module 1 실행 시 아래 파일이 생성된다.

- `outputs/module1/module1_overview.png`
- `outputs/module1/module1_simulation.gif`
- `outputs/module1/frames/step_00.png` ~ `step_05.png`

실행 명령:

```bash
cd /Users/kenny31/Documents/Capstone
bash scripts/run_module1.sh
```

## 그림이 의미하는 것
`module1_overview.png`는 마지막 step의 도시 상태를 한 장으로 보여준다.

- 바깥 큰 격자: 3x3 도시 블록
- 각 칸 안의 아이콘: 해당 블록에 있는 객체
- 오른쪽 상단 패널: 최근 이동 이벤트
- 오른쪽 하단 패널: 아이콘 종류 설명
- 상단 요약 바: 주요 객체 수

즉 이 그림은 "현재 어떤 블록에 승객, 택시, 차량, 장애물이 있는가"를 빠르게 보여주는 도시 스냅샷이다.

## 아이콘 해석
- 파란 원: 승객
- 노란 사각형: 택시
- 빨간 마름모: 자율주행 차량
- 검은 X: 장애물
- 초록/보라/갈색/하늘색 삼각형: 일반 차량 종류

## 칸 위의 작은 요약 태그
각 칸의 왼쪽 위에 있는 짧은 텍스트는 해당 칸에 어떤 객체가 몇 개 있는지를 압축해서 보여준다.

예:
- `T1 P2` = 택시 1대, 승객 2명
- `SE2 SU1` = sedan 2대, SUV 1대
- `AV1 X1` = 자율주행 차량 1대, 장애물 1개

## GIF는 어떻게 보나
`module1_simulation.gif`는 step 0부터 step 5까지의 변화를 이어서 보여준다.

- step 0: 초기 배치
- step 1~5: 차량 이동 반영

이 GIF를 보면 "정적 배치"가 아니라 "시간에 따라 바뀌는 도시 상태"라는 점을 쉽게 설명할 수 있다.

## 이 시각화가 프로젝트에서 하는 역할
이 Module 1 시각화는 아직 Unity 3D 장면은 아니지만, 프로젝트 전체에서 아래 역할을 한다.

1. 디지털 트윈 최소 구현의 증거
2. 승객/택시/차량/장애물이 한 도시 안에 존재한다는 구조 설명
3. 이후 Unity 장면으로 확장할 기준 화면 제공
4. 발표 시 "현재 도시 상태를 어떻게 보여줄지"에 대한 예시 제공

## 현재 한계
- 실제 도로망 경로 탐색 없음
- 승객 호출 이벤트와 배차 결과 연동 없음
- 택시 승하차 로직 없음
- Unity 실시간 씬 연동 없음

즉 지금 시각화는 "발표 가능한 2D 디지털 트윈 초안"으로 보면 된다.

## 발표 때 이렇게 설명하면 된다
- Module 1은 예측과 배차 알고리즘이 적용되는 도시 환경을 시각적으로 표현하는 단계입니다.
- 현재는 Python 기반 2D 디지털 트윈 시각화를 통해 객체 배치와 이동 이벤트를 확인할 수 있도록 구성했습니다.
- 이후 Unity 기반 3D 장면으로 확장하여 배차 결과를 더 직관적으로 보여줄 계획입니다.
